# Issue-Übersicht

Diese Datei ist die einzige Quelle für die Liste, den Status, die Priorisierung
und die Reihenfolge der Issues.

## Priorisierung

Die nächste Umsetzung wird nach folgender Reihenfolge bestimmt:

1. Status `offen`
2. Priorität `P0` bis `P4`
3. Issue-ID aufsteigend

Abgeschlossene Issues bleiben zur Dokumentation erhalten und werden nicht für die nächste Umsetzung berücksichtigt.

Backlog-Issues werden erst berücksichtigt, wenn sie ausdrücklich wieder aktiviert werden.

Die Priorität bleibt eine globale Einordnung. Für zusammenhängende
plattformbezogene Backlogs dokumentiert das Metadatenfeld
`<Plattform>-Phase` die verbindliche Reihenfolge innerhalb dieses Backlogs;
es verändert die Priorisierungslogik nicht.
